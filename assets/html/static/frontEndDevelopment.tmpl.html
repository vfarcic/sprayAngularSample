Front-end: Development

Front-end development is decoupled from the server. However, we still need to serve static files (HTML, JavaScript, CSS, images...) so it needs to be running as explained in the Development: running the application section. Gulp will make sure that every time there is any change to the code, all tasks are executed and our change is visible in the browser. 

gulp watch

Feel free to change any HTML or JavaScript code. As soon as the change is saved, Gulp will re-run all tasks. In case of an error, Gulp will worn us about them. After Gulp is run successfully, open the screen affected by the change to review them.

Following tasks are run as part of gulp watch:

  • jshint
  • concat
  • ngAnnotate
  • uglify
  • sourcemaps
  • jasmine
There is a brief description of each of those tasks. Their execution is very fast and, since they are part of gulp watch, they will all run on every change to the code.

JSHint

JSHint is a community-driven tool to detect errors and potential problems in JavaScript code and to enforce your team's coding conventions.

Concat

Gulp will concatenate all JavaScript files into app.js. This way there is no need to include each new JavaScript file into the index.html file. It is enough to keep the app.js file referenced. Following is the snipped from the index.html file. 
<head>
    <script src="/assets/app.js"></script>
</head>

NG Annotate

NG Annotate adds angularjs dependency injection annotations. Without it, our code would not work after it gets minified.

Uglify

Uglify minifies JavaScript files. Minification is the process of removing all unnecessary characters from source code without changing its functionality. Minified code is significantly smaller and results in reduced amount of data that needs to be transferred to the client ultimately making web pages load faster.

Source maps

After the JavaScript code is concatenated and minified, it cannot be debugged without having a relation with original code. Source maps fix that problem.

Jasmine

Jasmine is a behavior-driven development framework for testing JavaScript code. We're using it for all our front-end unit tests. The test code resides in the test/html directory. Structure of that directory mimics the assets/html so that the relation between the implementation code and tests is clearly established.
